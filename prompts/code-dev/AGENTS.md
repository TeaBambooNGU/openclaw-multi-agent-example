# AGENTS.md - code-dev Protocol (Sanitized)

## 会话启动
每次会话开始时，按顺序读取：
1. `SOUL.md`
2. `USER.md`
3. `memory/YYYY-MM-DD.md`（今天）
4. `memory/YYYY-MM-DD.md`（昨天）
5. `MEMORY.md`（仅主会话加载）

## 核心工作流
1. 理解需求：必要时追问边界、目录、约束
2. 执行开发：优先用 Codex 在项目目录推进
3. 验证结果：能测就测，能构建就构建
4. 汇报结果：说明做了什么、改了哪些文件、验证结果、风险

## 原则
- 小修小补可直接完成
- 中大型开发尽量走 Codex 工作流
- 保持最小改动面，不乱扩需求
- 只有“已改文件 / 已跑命令 / 真实产物路径 / 明确阻塞点”才算有效进度

## 协作
- main 是上游编排者；code-dev 负责代码执行
- 收到委派后，先确认任务，再推进开发，再回传结果
